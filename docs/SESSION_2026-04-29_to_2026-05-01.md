## 3. Architecture Decision — Two-Stage Handoff vs Unified Model

Discussed whether to train a single unified model with two heads (gem species + price) vs maintaining the two-model pipeline.

**Unified model approach:** Shared backbone, Head 1 predicts gem species (68 classes), Head 2 predicts diamond price. Price loss only backpropagates on diamond samples during training.

**Decision: two-stage handoff.**

Rationale:
- Price head only gets gradient from diamond samples — sparse relative to Head 1's 68 classes
- Loss weighting between tasks is non-trivial to tune
- Independent models are easier to retrain, benchmark, and debug
- Stage 1 at 82% diamond accuracy is strong enough that cascade loss is acceptable
- The two objectives (gem identity vs price) are sufficiently different that a shared backbone would be a representational compromise for both

## 4. Combined Dataset — Motivation and Design

### Motivation

The combined dataset trains Stage 2 on data from both retailers simultaneously. Previously all Stage 2 models were trained on one retailer and one diamond type at a time (ja_natural, be_lab, etc.). Cross-domain results showed F1 collapses from ~0.65 within-site to 0.07–0.31 when testing across retailers, so the model must see both retailers during training.

The correct Stage 2 model for the full pipeline must handle:
- Both retailers (JamesAllen + BrilliantEarth)
- Both diamond types (natural + lab)
- Because Stage 1 only outputs "Diamond" — it does not distinguish natural vs lab

This makes `combined_all` the correct dataset for the final pipeline model.

### Split design

Three combined subsets built by `scripts/build_combined_splits.py`:

| Subset | Train | Val | Test | Contents |
|---|---|---|---|---|
| combined_natural | 149,730 | 32,084 | 32,197 | ja_natural + be_natural |
| combined_lab | 170,122 | 36,456 | 57,360 | ja_lab + be_lab |
| combined_all | 299,460 | 64,168 | 89,557 | all four subsets |

**Key design decisions:**
- Train/val are subsampled to the smallest contributing site to prevent retailer imbalance
- Test sets are **not** subsampled — all images from all sites included for realistic evaluation
- Log-price is z-scored per source subset using train-split stats, then applied to val/test — no leakage
- Tier labels kept as-is (already represent relative value within each subset's distribution)

**Test tier distribution (combined_all):**

| Tier | Count | Share |
|---|---|---|
| mid_range | 43,492 | 48.6% |
| budget | 23,245 | 26.0% |
| premium | 13,160 | 14.7% |
| investment_grade | 9,660 | 10.8% |

Mid-range dominates — reflects real-world price distribution. Macro F1 penalizes equally across tiers so investment_grade performance pulls the overall number down.

## 7. Key Findings

1. **EfficientNetV2 is the Stage 2 model.** Leads classification and regression across every dataset and subset combination.

2. **Combined training beats single-site.** Combined_natural F1=0.6979 vs best single-site 0.6724. More data from both retailers provides meaningful improvement.

3. **Natural diamonds are more predictable than lab.** Consistent gap across all architectures and tasks. Lab diamond pricing has less visual signal.

4. **Cross-domain training is mandatory.** F1 collapses to 0.07–0.31 when training on one retailer and testing on the other. The Stage 2 pipeline model must train on combined_all.

5. **combined_all is the correct pipeline model, not combined_natural.** Despite lower metrics, it is the only model that handles both natural and lab diamonds from both retailers — which is what the full pipeline will receive at inference time.

6. **Regression Spearman scores are strong.** 0.93 on ja subsets — price ranking is highly reliable even where absolute USD error is high.

## 8. Pipeline Evaluation Script

`src/evaluate_pipeline.py` written to evaluate the full two-stage pipeline end-to-end.

### Test set composition

| Source | Images | Notes |
|---|---|---|
| combined_all test split | 89,557 | All diamond images — natural + lab, ja + be |
| Combined-P1-Dataset test | 1,767 | Non-diamond images, 67 species |
| **Total** | **91,324** | |

### Metrics reported

**Stage 1:**
- Diamond recall — percentage of true diamonds correctly identified as Diamond
- False positive rate — percentage of non-diamonds incorrectly called Diamond

**Stage 2 (on Stage-1-passed images only):**
- Price tier accuracy and macro F1
- Per-class classification report (budget / investment_grade / mid_range / premium)
- USD price mean absolute error
- Median absolute percentage error in USD

**End-to-end:**
- Tier accuracy where a Stage 1 miss counts as a wrong answer
- Per-subset breakdown (ja_natural, ja_lab, be_natural, be_lab)

### USD price recovery

Model outputs normalized log-price. Denormalized back to USD using per-subset mean and std computed from the training split:

| Subset | Log-price mean | Log-price std |
|---|---|---|
| be_lab | 8.0103 | 0.9914 |
| be_natural | 7.7412 | 1.0465 |
| ja_lab | 7.6527 | 0.8432 |
| ja_natural | 7.6878 | 1.2527 |

Normalization stats are recomputed at eval time from `combined_all_train.csv` — not saved separately.
