## Environment Setup

### PyTorch CUDA Reinstall

The session note from March 30 flagged that PyTorch was installed as CPU-only in
`ja_scraper/venv`. Reinstall was attempted using the specified `cu121` index URL:

```bash
python3 -m pip install torch torchvision \
    --index-url https://download.pytorch.org/whl/cu121 --upgrade
```

**Issue encountered:** RTX 5070 Ti (Blackwell, sm_120) is not supported by `cu121`.
PyTorch cu121 tops out at sm_90 (Ada Lovelace / RTX 4000 series). The following warning
confirmed the mismatch:

```
NVIDIA GeForce RTX 5070 Ti with CUDA capability sm_120 is not compatible with the
current PyTorch installation. The current PyTorch install supports CUDA capabilities
sm_50 sm_60 sm_70 sm_75 sm_80 sm_86 sm_90.
```

**Resolution:** Switched to `cu128` index URL — the first PyTorch build with Blackwell
support:

```bash
python3 -m pip install torch torchvision \
    --index-url https://download.pytorch.org/whl/cu128 \
    --force-reinstall
```

**Verified:**
```
torch: 2.11.0+cu128
CUDA available: True
GPU: NVIDIA GeForce RTX 5070 Ti
CUDA: 12.8
```

**Note for future sessions:** Any machine with an RTX 5000 series (Blackwell) GPU must
use `--index-url https://download.pytorch.org/whl/cu128`. The `cu121` URL specified in
the March 30 session note is incorrect for protouno.

### Bugs Fixed During Bring-Up

Three bugs were found and patched before training started:

1. **`python` not in PATH** — shell scripts used bare `python`. Fixed with `sed` to use
   absolute venv path (`/mnt/storage/projects/DLCGIPG/ja_scraper/venv/bin/python3`).

2. **`class_weights.json` structure mismatch** — `train.py` expected a list; actual
   format is `{"0": float, "1": float, "2": float, "3": float}`. Fixed:
   ```python
   weights_list = [all_weights[weight_key][str(i)] for i in range(4)]
   ```

3. **`get_dataloader()` signature mismatch** — `train.py` passed `class_weights=list`;
   actual signature takes `class_weights_path=path_to_json`. All three call sites patched
   to pass `class_weights_path=str(weights_path)` for train split; val/test calls drop
   the argument entirely (no sampler needed).

### CPU Thermals

| Sensor | Temp | Notes |
|---|---|---|
| Tctl | 83°C | Control temp (includes AMD offset) — drives fan/LED |
| Tccd1 | 80°C | Active die — DataLoader workers concentrated here |
| Tccd2 | 50°C | Second die largely idle |

CPU load is driven by 8 DataLoader workers doing JPEG decode + augmentation transforms
continuously. Tccd1 at 80°C is within spec (Tjmax ~95°C) but warm for a sustained run.
Monitor during longer ViT runs — reduce `NUM_WORKERS` to 6 if Tccd1 exceeds 88°C.

**Cooling:** Noctua air cooler (NH-D15 or equivalent) in pull-push configuration, fed
by 3× side intakes. Setup is well-configured. Thermal ceiling is cooler capacity vs
9900X sustained power draw — not an airflow problem.

**Key note:** `CUDA Version: 13.1` shown in nvidia-smi header is the driver's maximum
supported CUDA version, not the runtime version PyTorch uses (12.8). No conflict.
