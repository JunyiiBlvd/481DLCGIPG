## Key Bugs Fixed

### 1. JA Session Staleness (`scrape_v3.py`)

**Problem:** `requests.Session` cookies/state expired mid-run. JA's API rate limiter
returns an empty response body (HTTP 200, 0 bytes) instead of an error code. This caused
`json.JSONDecodeError: Expecting value: line 1 column 1 (char 0)`. The original code
silently skipped bands on this error, leaving gaps in the checkpoint.

**Fix:** In `call_api()`, on `JSONDecodeError`:
- Reset `_session = None` (force a fresh session on next call)
- Sleep `20 * (attempt + 1)` seconds (exponential, lets rate limit cool)
- Continue retry loop with fresh session

This is a root-cause fix: stale sessions self-heal within the same run rather than
requiring a kill/restart.

```python
except json.JSONDecodeError:
    global _session
    _session = None
    pause = 20 * (attempt + 1)
    _log(f"  JSON decode error — session reset, sleeping {pause}s")
    time.sleep(pause)
    s = get_session()
    continue
```

Also changed final `return None` → `return False` so all call failures propagate
uniformly as False, preventing accidental checkpointing of failed bands.

### 2. Stale Checkpoint Contamination

**Problem:** Old scrape runs had marked certain bands as "completed" in checkpoint JSON
files but with zero rows collected (session had gone stale during that band). Those bands
were then skipped on all subsequent runs.

**Fix:** One-time cleanup script: loaded each checkpoint, counted CSV rows per band key,
stripped any completed band with 0 rows (except bands above 4.0ct where 0 results is
genuinely correct). Had to run multiple times as fresh sessions also went stale.

---

### 3. BE Scraper CWD Mismatch

**Problem:** All scraper scripts use `Path("output")` (relative to CWD). Multiple times
the BE scraper was launched from `ja_scraper/` instead of `be_scraper/`, silently
writing data to the wrong site's output directory.

**Impact discovered:** One CWD error caused 2,468 BE rows to be appended to JA's
`diamonds_lab_oval_raw.csv`. Fixed by filtering on `image_url` domain and restoring
14,073 JA rows.

**Prevention:** Always verify CWD before launching scrapers. Use:
```bash
cd /mnt/storage/projects/DLCGIPG/be_scraper && python scrape.py ...
```

---

### 4. Redundant Lab Downloader

**Problem:** A `download_images.py` process was launched with
`--csv output/diamonds_lab_raw.csv` (no tier labels). Since the raw CSV has no
`value_tier` column, all 45,085+ images were written to `unlabeled/` as duplicates
of images already in tier folders.

**Fix:** Killed the process. Removed 45,085 duplicate images from `unlabeled/`
(identified as any file in `unlabeled/` that also existed in a tier folder).

---

### 5. JA Image CDN Patterns (3 wasted requests per image)

**Problem:** `download_images.py` had three CDN URL patterns hardcoded. Testing
confirmed the first two (`cdn1.jamesallen.com` variants) always returned HTTP 403.

**Fix:** Stripped to single pattern — use the scraped `image_url` directly:
```python
IMAGE_CDN_PATTERNS = ["{IMAGE_URL}"]
```
Eliminates 2 wasted requests per image, halved download time.

---

### 6. `label_tiers.py` Architecture (JA)

**Problem:** Original `label_tiers.py` read a single `diamonds_raw.csv` and computed
one set of percentile boundaries for all diamonds. Lab-grown diamonds occupy a
fundamentally different price space — mixing them with naturals for boundary computation
produces meaningless tiers.

**Fix:** Rewrote `label_tiers.py` to:
- Load `diamonds_natural_raw.csv` and `diamonds_lab_raw.csv` separately
- Compute price boundaries independently per subset
- Assign tiers, add `site = "james_allen"` column
- Merge and write `diamonds_labeled.csv`

This matches the `be_labeler.py` pattern used on the BE side.

---

### 7. `print_report()` Signature Mismatch

**Problem:** Updated `main()` called `print_report(all_rows, nat_bounds, lab_bounds)`
but old function signature was `print_report(rows)` (single arg). Script crashed
mid-run after successfully normalizing 16,771 shape names, before writing the CSV.

**Fix:** Updated `print_report()` to accept `(rows, nat_bounds, lab_bounds)`.

---

### 8. `tier_stats.json` KeyError

**Problem:** A round-folding script tried to read nat/lab boundaries from
`tier_stats.json` using keys `stats['boundaries']['natural']['p25']`. The actual file
written by `label_tiers.py` stores a flat structure: `stats['boundaries']['p25']`
(only one boundary set, whichever ran last).

**Fix:** Computed boundaries directly from the raw CSV price data at script runtime
instead of reading from the JSON stats file.

---

### 9. JA Shape Name Normalization

**Problem:** JA API returns "Cushion Modified" and "Square Radiant" as distinct shape
names. BE only has "cushion" and "radiant". Cross-domain experiments require matching
shape vocabularies.

**Fix (applied in final cleanup script):**
- 16,771 rows: "Cushion Modified" → "cushion"
- 408 rows: "Square Radiant" → "radiant"

Applied directly to `diamonds_labeled.csv` as part of the round-folding pass.

---

### 10. `download_watchdog.sh` CWD Bug

**Problem:** Watchdog set `SCRIPT="be_scraper/download_images.py"` and ran from project
root. The downloader uses `Path("output")` (relative), which resolved to
`/DLCGIPG/output/` instead of `/DLCGIPG/be_scraper/output/`. Error:
`diamonds_natural_raw.csv not found`.

**Fix:** Added `cd "$PROJECT_ROOT/$SCRIPT_DIR"` before running the download script,
with `VENV_PYTHON` and `SCRIPT` path adjusted accordingly.

## New Scripts Added

### `ja_scraper/watchdog.sh`

Runs `scrape_v3.py` and restarts up to 6 times until all shape checkpoints show 575
completed bands. Checks via embedded Python that reads each shape's
`scrape_v3_{prefix}_{shape}_checkpoint.json`.

```bash
bash watchdog.sh natural oval,cushion,pear,princess,emerald,marquise,asscher,radiant,heart /tmp/ja_nat.log
bash watchdog.sh lab     oval,cushion,pear,princess,emerald,marquise,asscher,radiant,heart /tmp/ja_lab.log
```

### `download_watchdog.sh` (project root)

Generic download watchdog for both sites. First pass downloads everything; subsequent
passes run `--missing` only. Retries up to 5 times with 60s pause (CDN rate limit
cooldown). Checks `download_failures.csv` row count to determine completion.

```bash
bash download_watchdog.sh ja                        # JA natural
bash download_watchdog.sh be                        # BE natural
bash download_watchdog.sh be --lab                  # BE lab
```

## Scraper Architecture Reference

### James Allen (`ja_scraper/scrape_v3.py`)

- Transport: `requests` (plain HTTP, no browser)
- API: JA internal JSON endpoint, queried with carat band `(cmin, cmax)` per shape
- Sweep: 0.25–6.00ct in 0.01ct steps = 575 bands per shape
- Checkpoint: `scrape_v3_{prefix}_{shape}_checkpoint.json` — keyed by `(cmin, cmax)` float tuple
- Session refresh: resets on JSONDecodeError, sleeps before retry
- Shapes: oval, cushion, pear, princess, emerald, marquise, asscher, radiant, heart, round
- Output: `diamonds_natural_raw.csv`, `diamonds_lab_raw.csv` (per shape, then merged)

### Brilliant Earth (`be_scraper/scrape.py`)

- Transport: Playwright (headless Chromium) — required for Cloudflare JS challenge
- API: BE internal search endpoint captured via network intercept
- Sweep: carat band micro-steps (similar pattern to JA)
- Checkpoint: `scrape_{natural|lab}_{shape}_checkpoint.json`
- Output: `diamonds_natural_raw.csv`, `diamonds_lab_raw.csv`

### Labeling Pipeline

1. `label_tiers.py` (JA) / `be_labeler.py` (BE) — compute percentile boundaries
   independently for natural and lab subsets, assign `value_tier`, write labeled CSV
2. Tier boundaries: P25 = budget cutoff, P75 = mid-range cutoff, P90 = premium cutoff
3. `download_images.py` — fetch images, skip already-downloaded (resume-safe)
4. `download_watchdog.sh` — retry wrapper with `--missing` passes
