## Dataset Remediation — BE Round Gap

### Root Cause

BE scraper ran per-shape, producing individual per-shape CSVs. A merge step combined
them into `diamonds_natural_raw.csv` / `diamonds_lab_raw.csv`. Round was scraped in a
separate session and never included in that merge. Everything downstream inherited the
gap silently:

```
per-shape CSVs (all 10 shapes, including round)
        ↓
merge step ← round excluded here
        ↓
diamonds_natural_raw.csv / diamonds_lab_raw.csv (9 shapes only)
        ↓
be_labeler.py → be_labeled.csv (9 shapes only)
        ↓
image downloader → images/ (9 shapes only)
```

JA had the same structural issue but was accidentally rescued — `diamonds_labeled.csv`
accumulated across multiple scraping sessions and captured round. BE's labeled CSV was
built cleanly from the merged raws in one shot, so round was permanently excluded.

**Prevention:** Validate shape coverage immediately after any merge step before running
anything downstream.

### Remediation Steps

1. Confirmed `diamonds_natural_round_raw.csv` and `diamonds_lab_round_raw.csv` exist
   as per-shape artifacts in BE scraper output
2. Applied tier boundaries from `be_tier_stats.json` to round rows (no recomputation —
   boundaries held from existing 9-shape inventory)
3. Appended to `be_labeled.csv` → `be_labeled_v2.csv` (367,946 rows)
4. Extracted round-only rows to `be_round_only.csv` (104,509 rows) — fed to downloader
   to avoid triggering re-download of all 263k existing images (skip logic checks flat
   `images/{diamond_id}.jpg`, not tier subdirs; passing full v2 CSV would re-download everything)
5. Downloaded 104,495 / 104,509 round images (14 failures: 13× HTTP 403, 1× 1.8KB broken)
   — failures logged to `be_round_download_failures.csv`
6. Dropped 14 round failure rows → `be_labeled_v3.csv` (367,932 rows)
7. Organized 104,495 round images into tier subdirectories using `organize_images()` logic
   (replicated inline — not called via `be_labeler.py` main() to avoid recomputing boundaries)
8. Dropped 107 pre-existing missing image rows (non-round, carry-over from original scrape,
   spread across 9 shapes) → `be_labeled_final.csv` (367,825 rows)
9. Verified zero orphans, zero CSV/image delta ✅

**Intermediate files (all in `be_scraper/output/`):**
- `be_labeled_v2.csv` — 367,946 rows (after appending round)
- `be_round_only.csv` — 104,509 rows (temp, used for targeted download)
- `be_round_download_failures.csv` — 14 rows (renamed from `download_failures.csv`)
- `be_labeled_v3.csv` — 367,932 rows (after dropping 14 failures)
- `be_labeled_final.csv` — 367,825 rows (after dropping 107 pre-existing missing)

**Tier boundary disclosure:** BE round tier labels were assigned using boundaries derived
from non-round inventory (9 shapes). If re-run from scratch on the full 10-shape dataset,
boundaries would shift slightly as round diamonds price at a premium over fancy shapes.
Labels are internally consistent but carry this approximation. Disclose in paper.

## Preprocessing Pipeline

All outputs at `/mnt/storage/projects/DLCGIPG/data/`

### Step 1 — CSV Cleaning (`processed/`)

**Both datasets:**
- Dropped `cert_number` (100% null)
- Normalized shape labels to lowercase
- Remapped cut: `"Excellent"` (JA) → `"Ideal+"`, `"Super Ideal"` (BE) → `"Ideal+"`
- Added `tier_label` integer column: `budget=0, mid_range=1, premium=2, investment_grade=3`
- Added `is_lab` boolean column

Output: `ja_cleaned.csv` (229,204 rows, 26 cols), `be_cleaned.csv` (367,825 rows, 25 cols)

### Step 2 — Natural/Lab Splits (`processed/`)

Split each cleaned CSV by `is_lab`:

| File | Rows |
|---|---|
| `ja_natural.csv` | 107,687 |
| `ja_lab.csv` | 121,517 |
| `be_natural.csv` | 106,950 |
| `be_lab.csv` | 260,875 |

**Notable distributions:**
- BE natural skews heavily round (43,710 / 41%) — round is BE's dominant natural shape
- BE lab: 71% of BE dataset vs JA's 53% — systematic, not random
- JA natural/lab: ~53/47; BE natural/lab: ~29/71

### Step 3 — Train/Val/Test Splits (`splits/`)

Random seed 42. All splits stratified by `tier_label`.
`be_natural` additionally stratified by `tier_label × shape` (40 groups, 0 rare combos)
to prevent round dominance (41% of be_natural) from skewing any single split.

**Within-site (70/15/15):**

| Split | Train | Val | Test |
|---|---|---|---|
| ja_natural | 75,380 | 16,153 | 16,154 |
| ja_lab | 85,061 | 18,228 | 18,228 |
| be_natural | 74,865 | 16,042 | 16,043 |
| be_lab | 182,612 | 39,131 | 39,132 |

**Cross-domain (80/20 train/val — full other site is test):**

| Split | Train | Val | Test |
|---|---|---|---|
| ja_natural | 86,149 | 21,538 | full BE natural |
| ja_lab | 97,213 | 24,304 | full BE lab |
| be_natural | 85,560 | 21,390 | full JA natural |
| be_lab | 208,700 | 52,175 | full JA lab |

**Splits are locked.** Paired `_ids.json` files (ordered `diamond_id` lists) saved alongside
each CSV for full reproducibility. Load `_ids.json` to reconstruct any split without
re-running the script.

All 20 split CSVs + 20 `_ids.json` files + `splits_summary.csv` are in `data/splits/`.

### Step 4 — Class Weights (`splits/class_weights.json`)

Computed per training split: `weight[class] = total / (num_classes × class_count)`

- Imbalance consistent across all subsets (~4–5×)
- `investment_grade` always minority (weight ~2.2–2.5×), `mid_range` always majority (weight ~0.5×)
- Cross-train weights used as canonical per subset (larger n = more stable estimate)
- Within-site and cross-train weights differ only at 6th decimal place — functionally identical

### Step 5 — PyTorch Dataset (`src/diamond_dataset.py`)

- `DiamondDataset(csv_path, image_dir, transform=None)`
- Image path: `image_dir / value_tier / f"{diamond_id}.jpg"`
- Returns `(image_tensor, tier_label)`
- Missing images filtered at `__init__` (not `__getitem__`) — authoritative length, no None collation issues
- `labels` property exposes full label list for WeightedRandomSampler without image I/O

**Transforms:**
- `train_transform`: Resize 224×224, RandomHorizontalFlip, RandomRotation(15°),
  ColorJitter(brightness=0.2, contrast=0.2, saturation=0.1, hue=0.05), ToTensor, ImageNet normalize
- `val_test_transform`: Resize 224×224, ToTensor, ImageNet normalize

**`get_dataloader()` helper:** Returns DataLoader with WeightedRandomSampler for train
splits, standard sampler for val/test.

**Smoke test:** Passed. WeightedRandomSampler confirmed working — first batch of 32
samples: `{0:9, 1:6, 2:8, 3:9}` (vs expected ~8× more class 1 without resampling).
Pixel range [-2.03, 1.79] — correct for ImageNet-normalized tensors.

## RF Baseline Results (`results/rf_baseline/`)

RandomForestClassifier(n_estimators=300, class_weight="balanced", random_state=42, n_jobs=-1)
Features: carat, cut, color, clarity (ordinally encoded, fit on train only)

**Ordinal encoding (worst=0, best=max index — fixed domain-knowledge map, not data-fitted):**
- cut: Fair=0, Good=1, Very Good=2, Ideal=3, Ideal+=4
- color: K=0, J=1, I=2, H=3, G=4, F=5, E=6, D=7
- clarity: I1=0, SI2=1, SI1=2, VS2=3, VS1=4, VVS2=5, VVS1=6, IF=7, FL=8

OOV grades (e.g. "Fair" in JA which has no "Fair" in some splits) mapped to -1.

| Split | Test Accuracy | Macro F1 | Carat Importance |
|---|---|---|---|
| ja_natural | 0.8658 | 0.8567 | 0.853 |
| ja_lab | 0.9411 | 0.9217 | 0.793 |
| be_natural | 0.8371 | 0.8226 | 0.805 |
| be_lab | 0.8961 | 0.8742 | 0.784 |

**These are the performance ceiling numbers for Stage 2 image models.**

**Carat dominance (0.79–0.85) is expected, not a labeling artifact.** Diamond price is
legitimately driven by carat more than any single other factor. The 15–21% signal from
color/clarity/cut confirms price-percentile labeling correctly reflects that grade quality
matters too. A pure labeling artifact would show carat importance ~1.0 and F1 ~1.0.

**Pre-registered prediction for paper:** `premium` is the hardest class across all splits
(F1 0.76–0.86). It sits between two larger adjacent classes with the least carat separation
from `mid_range`. If confusion matrices from image models confirm this, it validates the
prediction from first principles.

**JA lab (F1=0.92) is the easiest subset** — lab diamonds have tighter grade distributions
and more predictable pricing by carat. If image models show the same ordering (JA lab
easiest, BE natural hardest), that's consistent evidence the task difficulty is real and
not a modeling artifact.

## Key Design Decisions Logged

- **Canonical class weights:** Cross-train version used for all experiments per subset
  (larger n, more stable estimate). Same weights apply to both within-site and cross-domain
  runs of the same subset.
- **BE image suspect threshold:** 3KB (not 10KB). BE serves 300×300 grey-background
  thumbnails that compress efficiently. 10KB threshold was calibrated for JA's 757×600
  files.
- **Shape case normalization:** All shape labels lowercase. BE was Title Case; JA was
  lowercase. Any pipeline joining or comparing shape labels must normalize case.
- **Join key:** Both datasets use `diamond_id` (integer) as the image filename. Full path:
  `{image_dir}/{value_tier}/{diamond_id}.jpg`
- **`__init__` vs `__getitem__` filtering:** Missing images filtered once at dataset
  construction for authoritative length and no None collation errors.
